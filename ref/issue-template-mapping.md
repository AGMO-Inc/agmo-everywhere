# 이슈 템플릿 매핑 (SSOT: AGMO-Inc/.github)

## 템플릿 확인

```bash
gh api "repos/AGMO-Inc/.github/contents/.github/ISSUE_TEMPLATE?ref=main"
```

## Feature (`issue-type: feature`)

- 제목: `[Feature] {제목}`
- 템플릿: `01-기능-개발.yml`
- 섹션: `1. 한 줄 요약`, `2. 배경/문제`, `3. 요구 사항`, `4. 작업 항목`, `5. 참고`, `6. 수용 기준 (선택)`, `7. 상태 모델 (선택)`, `8. API/프로토콜 (선택)`, `9. 데이터 모델 (선택)`, `10. 비기능 요구사항 (선택)`

## Task (`issue-type: task`)

- 제목: `[Task] {상위 기능} - {제목}`
- 템플릿: `02-기능-개발---하위-태스크.yml`
- 섹션: `상위 Feature`, `1. 작업 요약`, `2. 체크리스트`, `3. 참고 (선택)`, `4. 상세 수용 기준 (선택)`, `5. 검증 로그/링크 (선택)`

## Bug (`issue-type: bug`)

- 제목: `[Bug] {제목}`
- 템플릿: `03-버그-리포트.yml`
- 섹션: `1. 증상 한 줄 요약`, `2. 기대 동작`, `3. 실제 동작`, `4. 재현 방법 (Optional)`, `5. 빈도`, `6. 사용자 영향`, `7. 해결 방법`
